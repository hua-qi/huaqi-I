# Huaqi 项目路线图

> 分阶段实施策略 - 从 MVP 到完整形态

---

## 总体策略

**原则**: 每个 Phase 产出可用的产品，逐步叠加能力

```
Phase 1 ──> Phase 2 ──> Phase 3 ──> Phase 4 ──> Phase 5
 (基础)     (记忆)     (智能)     (技能)     (生态)
 可用       好用       聪明       能干       强大
```

---

## Phase 1: 基础架构 (Week 1-2)

**目标**: 搭建可运行的脚手架，能对话、能配置

### 交付物
- [ ] 项目基础结构
- [ ] 配置管理系统
- [ ] CLI 框架
- [ ] 基础对话能力

### 任务清单

| 优先级 | 任务 | 验收标准 |
|--------|------|----------|
| P0 | 项目初始化 | `huaqi --version` 可执行 |
| P0 | 配置管理 | `huaqi config init` 成功创建配置 |
| P0 | CLI 框架 | Typer + Rich 集成完成 |
| P1 | LLM 接口抽象 | 支持至少一个 LLM 提供商 |
| P1 | 基础对话 | `huaqi chat` 可以对话 |
| P2 | 错误处理 | 友好的错误提示 |
| P2 | 日志系统 | 分级日志记录 |

### 技术产出

├── huaqi/
│   ├── __init__.py
│   ├── core/
│   │   ├── config.py
│   │   └── llm.py
│   └── interface/
│       └── cli/

├── requirements.txt
└── pyproject.toml
```

## Phase 2: 记忆系统 (Week 3-4)

**目标**: 实现混合存储方案，能记住关键信息

### 交付物
- [ ] 三层记忆实现
- [ ] 文件 + 向量混合存储
- [ ] 记忆自动提取
- [ ] Git 同步机制

### 任务清单

| 优先级 | 任务 | 验收标准 |
|--------|------|----------|
| P0 | 文件存储层 | Markdown/YAML 读写 |
| P0 | 向量存储层 | Chroma 集成 |
| P0 | Embedding | 支持 OpenAI/本地模型 |
| P1 | 记忆模型 | Session/Working/Long-term 三层 |
| P1 | 记忆提取 | 对话后自动提取关键信息 |
| P1 | 记忆检索 | 语义搜索可用 |
| P2 | Git 同步 | `huaqi sync push/pull` |
| P2 | 数据迁移 | `huaqi export/import` |

### 技术产出
```
huaqi/memory/
├── storage/
│   ├── file_store.py
│   └── vector_store.py
├── layers/
│   ├── session.py

## 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| API 成本过高 | 高 | 支持本地模型、用量控制 |
| 记忆质量差 | 中 | 人工审核 + 反馈机制 |
| 同步冲突 | 中 | 时间戳 + 冲突解决策略 |
| 隐私顾虑 | 高 | 本地优先、加密存储 |
