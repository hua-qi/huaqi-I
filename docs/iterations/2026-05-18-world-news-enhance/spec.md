# Spec: world-news-enhance

> 优化世界新闻采集定时任务的产出内容：中英对照、链接+摘要、分类建议。

## 1. 要解决的问题

当前世界新闻采集任务（`huaqi world fetch`）产出的 `world/{date}.md` 文件存在三个问题：
1. **缺少原文链接**：RSS 条目的 URL 存在 metadata 中，未渲染到内容，用户无法点击查看原文
2. **摘要质量不可控**：英文新闻仅靠 LLM 翻译扩展，没有约束输出结构（链接+摘要），且不保留英文原标题，用户丢失原文信息
3. **无行动指导**：信息堆砌，没有为用户提炼「哪些值得重点关注」，用户需要自己逐条判断

## 2. 功能范围

**包含：**
- 新闻内容中英对照（英文原标题 → 中文标题 + 中文摘要）
- 每条新闻强制输出原文链接
- 每条新闻 2-3 段中文摘要，补充关键背景
- 末尾「重点关注建议」板块，按领域分类（AI/科技、宏观经济、行业动态等），每类列出值得关注的新闻及理由
- 优化报告系统（WorldProvider）的展示逻辑，使重点关注建议能被晨报/日报有效展示

**不包含：**
- 修改 RSS 源列表（保持现有 6 个源）
- 修改 GitHub Actions 调度频率或通知机制
- 修改 search_worldnews_tool 的搜索逻辑
- 新增新闻分类或标签系统（仅在 LLM prompt 层面做分类建议）
- 修改 Pipeline 以外的内容管道（capabilities/pipeline/）

## 3. 技术选型

**个性化重点关注判断**：使用 TELOS 用户画像。在增强 prompt 中注入用户画像（信念、目标、挑战、所学等），让 LLM 以「这条新闻和用户有什么关系」为标准判断关注优先级，而非泛泛的新闻重要性排序。

**整体方案**：RSS 加链接 + 增强 prompt 注入用户画像 + WorldProvider 优先展示建议。不改架构。

## 4. 验收标准

- [ ] AC-1: RSS 源采集的每条新闻 doc.content 包含原文链接（`**链接**：{url}`）
- [ ] AC-2: 增强后的 world 文件中每条新闻包含：英文原标题、中文标题、原文链接、2-3 段中文摘要
- [ ] AC-3: 增强后的 world 文件末尾包含「重点关注建议」板块，按领域分类列出值得关注的新闻及关注理由
- [ ] AC-4: WorldProvider 优先展示「重点关注建议」内容（如果存在），而非简单截断前 1000 字符
- [ ] AC-5: 中文源的新闻（36氪、虎嗅、少数派）保留原文内容并扩展细节，不发散翻译
- [ ] AC-6: 边界情况 — 空文件、LLM 调用失败时，系统优雅降级，不崩溃

## 5. 依赖

- 依赖已有模块：`WorldNewsEnricher`、`WorldNewsStorage`、`RSSSource`、`WorldProvider`、`TelosManager`（用户画像）
- 后续 feature 无直接依赖本 spec

## 6. 风险与假设

- **假设**：LLM（gpt-4o-mini）能稳定按新 prompt 格式输出结构化内容
- **风险**：prompt 变长后 token 消耗增加，可能导致超时或成本上升。缓解：控制 prompt 长度，限制每条摘要字数
- **风险**：LLM 输出格式可能不稳定（代码块包裹、偶尔多输出前言）。缓解：已有 `_extract_markdown` 兜底逻辑
