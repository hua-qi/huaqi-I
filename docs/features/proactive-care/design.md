# 主动关怀与长期模式学习设计方案

## 1. 主动关怀触发系统

### 1.1 触发条件（多层过滤）

```
第一层：数据充足性检查
├── 日记数量 ≥ 3 篇
├── 动态维度记录 ≥ 5 条
└── 用户画像已建立

第二层：情绪状态检查（满足任一）
├── 连续 2 天日记情绪为负面
├── 最近 3 条对话情绪均为负面
├── 情绪强度持续 ≥ 7（高焦虑）
└── 关键词触发："累"、"烦"、"不想活"等

第三层：时机检查
├── 距离上次关怀 ≥ 24 小时
├── 当前时间在用户活跃时段
├── 用户 6 小时内未主动对话
└── 非深夜时段（22:00-08:00 不触发）

第四层：频率控制
├── 同一问题不重复关怀
├── 每周最多 3 次关怀
└── 用户可设置关怀偏好（关闭/仅紧急/正常）
```

### 1.2 关怀内容生成策略

| 场景 | 关怀策略 | 示例 |
|------|---------|------|
| 情绪低落 | 陪伴式倾听 | "感觉你最近不太开心，想聊聊吗？" |
| 工作压力 | 提供支持 | "看你最近工作很忙，需要我帮你整理思路吗？" |
| 持续焦虑 | 温和建议 | "你好像焦虑好几天了，试试深呼吸？" |
| 深夜情绪 | 次日关怀 | 推迟到第二天早上触发 |
| 重大事件 | 深度关怀 | 识别用户提到的重大事件后 24h 跟进 |

### 1.3 防骚扰机制

- **冷却期**：同类型关怀 48h 内不重复
- **用户反馈**：用户回复"没事"后降低该类触发敏感度
- **手动关闭**：用户可随时关闭关怀功能
- **渐进式**：新用户从轻度关怀开始，逐步个性化

## 2. 长期模式学习系统

### 2.1 模式识别维度

| 模式类型 | 识别方法 | 输出 |
|---------|---------|------|
| **情绪周期** | 时间序列分析 | 每周一情绪低落、月底焦虑等 |
| **话题偏好** | 话题频率统计 | 持续关注的主题及变化 |
| **行为模式** | 交互时间分析 | 活跃时段、回复间隔习惯 |
| **情绪触发器** | 关联分析 | 提到"A"后通常会焦虑 |
| **成长轨迹** | 长期趋势 | 情绪整体改善/恶化趋势 |

### 2.2 洞察生成

**周报（每周生成）**
```
📊 本周洞察

情绪概览：😊 整体积极（平均情绪分：6.5/10）
情绪波动：周三最低（工作压力），周末回升
持续关注：职业发展（提及 5 次）、健康（3 次）
新发现：你开始关注冥想，可能需要减压建议

💡 建议
- 工作日晚上尝试 10 分钟冥想
- 周末可以多安排户外活动
```

**月报（每月生成）**
```
📈 本月成长报告

情绪趋势：↑ 比上月提升 15%
高光时刻：3 次情绪峰值都与人际关系相关
关注转移：从"工作焦虑"转向"个人成长"
行为变化：日记频率提升（从每周 2 篇到 5 篇）

🎯 下月建议
- 继续保持冥想习惯
- 可以尝试制定职业发展计划
```

### 2.3 模式应用

- **预测性关怀**：识别到用户即将进入焦虑周期时提前安抚
- **个性化建议**：基于长期模式给出针对性建议
- **成长追踪**：可视化展示用户情绪成长曲线

## 3. 数据模型设计

### 3.1 关怀记录
```python
@dataclass
class CareRecord:
    record_id: str
    trigger_type: str      # 触发类型：emotion_low / work_stress / keyword
    trigger_reason: str    # 触发原因描述
    care_content: str      # 实际发送的关怀内容
    user_response: str     # 用户回复（如有）
    user_feedback: str     # 用户反馈：helpful / neutral / annoying
    created_at: datetime
    acknowledged: bool     # 用户是否回应
```

### 3.2 模式洞察
```python
@dataclass
class PatternInsight:
    insight_id: str
    insight_type: str      # emotion_cycle / topic_trend / behavior_pattern
    pattern_description: str
    confidence: float      # 置信度 0-1
    evidence: List[str]    # 证据列表
    generated_at: datetime
    valid_until: datetime  # 洞察有效期
```

## 4. 实现步骤

1. **创建 `proactive_care.py`** - 主动关怀核心类
2. **创建 `pattern_learning.py`** - 模式学习核心类
3. **创建 `insight_generator.py`** - 洞察生成器
4. **集成到 CLI** - 定时检查和触发
5. **添加配置选项** - 用户可自定义关怀偏好

---

## 5. 风险控制

- **隐私保护**：所有模式分析本地完成，不上传云端
- **用户控制**：用户可查看、删除、导出自己的模式数据
- **透明度**：告知用户我们在分析什么、为什么触发关怀
- **退出机制**：一键关闭所有主动功能
