# ADR-001: 初始架构设计决策

## 状态
- **日期**: 2026-03-24
- **决策**: 已采纳
- **作者**: 连子蒙

## 背景

本次对话旨在设计一个个人 AI 助手系统，需要满足以下核心需求：

1. **记忆**: 记录身份、偏好、项目、学习轨迹
2. **执行**: 信息收集、编码辅助、内容创作、学习辅导
3. **成长**: AI 随着了解深入而进化
4. **陪伴**: 答疑解惑、辅助决策、提升认知

## 约束条件

1. 可能换公司/换电脑，需支持跨设备迁移
2. 计划做成通用工具，技术方案需可复用
3. 分阶段逐步实现完整形态
4. 使用 Spec-Driven 开发模式
5. 使用 GitHub 维护项目

## 方案对比

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| 纯向量存储 | 语义检索强 | 不可读、难维护 | ❌ |
| 纯文件存储 | 可读、版本控制 | 检索弱 | ❌ |
| **混合方案** | 兼顾两者 | 架构复杂 | ✅ 采纳 |

## 决策内容

### 1. 存储架构: 混合方案 (文本 + 向量)

```
Layer 3: 云端备份 (GitHub Private / S3)     ← 跨设备同步
Layer 2: 向量索引 (Chroma/SQLite)           ← AI 语义检索  
Layer 1: 文本文件 (Markdown/YAML)           ← 人类可读、版本控制
```

**理由**:
- 文本文件保证可读性和版本控制
- 向量索引支持语义检索
- Git 原生支持跨设备同步

### 2. 整体架构: 7 层架构 (基于 PAI 理念)

参考 Daniel Miessler 的 Personal AI Infrastructure:

1. **智能层**: LLM 核心 + 持续学习
2. **上下文**: 三层记忆（会话/工作/长期）
3. **个性**: 可配置的 AI 性格
4. **工具**: MCP 技能生态
5. **安全**: 多层防御 + Constitution
6. **编排**: Hook 系统 + 自动化
7. **接口**: CLI 为主 + 语音 + Web

### 3. 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 语言 | Python 3.10+ | AI 生态丰富 |
| CLI | Typer + Rich | 现代化、交互友好 |
| 向量库 | Chroma | 本地优先、零配置 |
| 配置 | Pydantic Settings | 类型安全 |
| 同步 | Git | 版本控制、跨设备 |

### 4. 开发策略: Spec-Driven + 分阶段

```
Phase 1 (2周): 基础架构 - 能对话、能配置
Phase 2 (2周): 记忆系统 - 能记住、能同步
Phase 3 (2周): 智能核心 - 有个性、会主动
Phase 4 (2周): 技能生态 - 有技能、能做事
Phase 5 (4周): 高级功能 - 自动化、多端
```

## 项目产出
