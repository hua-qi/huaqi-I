# 主动关怀系统

基于用户情绪状态、日记内容和对话历史，智能触发关怀消息。

## 功能概述

主动关怀系统在用户需要时主动提供情感支持，通过分析用户的日记情绪、对话情绪和历史模式，识别可能需要关怀的时机。

## 触发条件

关怀触发采用多层过滤机制，确保既不错过真正的需求，也不造成打扰：

### 第一层：数据充足性检查
- 日记数量 ≥ 3 篇
- 动态维度记录 ≥ 5 条

### 第二层：频率控制
- 每天最多触发 1 次
- 每周最多触发 3 次
- 同一问题不重复关怀

### 第三层：时机检查
- 安静时段（22:00-08:00）不触发
- 用户 6 小时内未主动对话才触发

### 第四层：情绪状态检查（满足任一）

| 触发类型 | 条件 | 优先级 |
|---------|------|--------|
| 关键词预警 | "不想活"、"活着没意思"、"想死"、"自杀" | URGENT |
| 高风险关键词 | "撑不下去了" | HIGH |
| 持续情绪低落 | 连续 2 天日记情绪负面 | NORMAL |
| 高焦虑状态 | 最近 2 次焦虑强度 ≥ 7 | NORMAL |
| 日常压力 | 关键词："好累"、"好烦"、"压力好大" | LOW |

## 使用方式

### 命令

```bash
# 手动触发关怀检查
/care

# 查看关怀统计
/care status

# 查看当前配置
/care config

# 修改配置
/care set <key> <value>
```

### 配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| enabled | true | 是否启用关怀 |
| level | "normal" | 关怀级别：minimal/normal/intensive |
| max_per_day | 1 | 每天最多触发次数 |
| max_per_week | 3 | 每周最多触发次数 |
| quiet_hours_start | 22 | 安静时段开始（小时）|
| quiet_hours_end | 8 | 安静时段结束（小时）|
| min_silence_hours | 6 | 最小沉默时长（小时）|
| emotion_threshold | 0.3 | 情绪阈值 |
| anxiety_threshold | 7 | 焦虑阈值（1-10）|

### 示例

```bash
# 关闭关怀
/care set enabled false

# 设置每天最多2次
/care set max_per_day 2

# 设置焦虑阈值为6（更敏感）
/care set anxiety_threshold 6
```

## 自动触发

CLI 启动时会自动执行关怀检查：

```
$ huaqi chat

🌸 Huaqi - 个人 AI 同伴

🌸 Huaqi: 感觉你最近心情不太好，想聊聊吗？
（这是基于你最近状态的关怀）
```

## 关怀内容

根据触发类型和优先级，系统会选择不同的关怀话术：

| 场景 | 关怀示例 |
|------|---------|
| URGENT | "我注意到你好像遇到了很难的事情。我在这里，愿意听你说。" |
| HIGH | "感觉你最近压力很大，想聊聊吗？" |
| NORMAL | "看你最近情绪有点低落，我在这里陪着你。" |
| LOW | "今天感觉怎么样？" |

## 数据存储

关怀记录存储在：`~/.huaqi/memory/care_records.jsonl`

包含字段：
- trigger_type: 触发类型
- trigger_reason: 触发原因
- care_content: 实际发送的关怀内容
- user_response: 用户回复
- user_feedback: 用户反馈（helpful/neutral/annoying）

## 隐私说明

- 所有分析在本地完成，不上传云端
- 用户可随时关闭关怀功能
- 关怀记录可随时删除
